Abstract & Motivation

This document serves as a roadmap and a gift for the "Three Kings" party, honoring the foundational contributions of Tom Toffoli, Norm Margolus, and Charlie Bennett to digital physics and reversible computing. Conceived during a Friday car ride and realized over a 6-hour coding sprint on a Saturday morning, this experiment re-codes the principles of SIMP/STEP into a modern, tensor-accelerated framework.

The result is Generative Lattice Flow—a methodology that combines human architectural insight with Large Language Models (LLMs) to compile high-level physical rules into bare-metal tensor operations. By decoupling transport and interaction, we achieve a bit-perfect "Bennett Echo" in an HPP lattice gas, reaching a throughput of 1.47 GSPS (Giga-Sites Per Second) on a standard cloud CPU.

I. The LatticeFlow Conceptual Framework

The core of this methodology is the formal decoupling of Movement (where a particle goes) from Logic (what happens when particles meet). This absolute separation is mathematically necessary for functional reversibility in discrete physics.

Concept The Physics (What it represents) The Geometry (How it operates)
Transport (Shift) Convection / Momentum: The linear movement of a particle through space over a time step dt. No energy is exchanged. A translation of a bit-plane or signal by a vector v. A non-local operation where data moves across memory.
Interaction (Rule) Collision / Scattering: The local exchange of state between particles at the same spatial coordinate. Energy/Momentum are conserved here. A logic function f(s) applied to the "stack" of bits at a single site. A strictly local operation (no neighbor access needed).

III. Historical Lineage of the Kernel

A crucial outcome of this 6-hour sprint is translating the historic data movement paradigms into pure JAX Python. By tracing the evolution of the HPP Collision Rule, we can see how the conceptual "trick" remains the same even as the hardware changes.

1. CAM-8 (Hardware LUT Generator, ~1990s) C / Forth Context 
/* The CAM-8 pre-calculated all possible bit-states into an SRAM lookup table.
   This generator evaluates head-on collisions to build the table. */
int generate_hpp_lut(int state) {
    int n = state & 1, s = (state>>1)&1, e = (state>>2)&1, w = (state>>3)&1;
    if (n && s && !e && !w) return (1<<2) | (1<<3); // NS -> EW
    if (e && w && !n && !s) return 1 | (1<<1);      // EW -> NS
    return state; // Identity for all other states
}
2. SIMPSTEP (C-Macro Logic, ~2000s) C Kernel 
/* SIMPSTEP relied on virtual-interrupt scanning and C-based bit manipulation
   applied dynamically to individual grid sites. */
void hpp_rule(site_t *site) {
    int n = N(*site), s = S(*site), e = E(*site), w = W(*site);
    if (n && s && !e && !w) {
        N(*site)=0; S(*site)=0; E(*site)=1; W(*site)=1;
    } else if (e && w && !n && !s) {
        E(*site)=0; W(*site)=0; N(*site)=1; S(*site)=1;
    }
}
3. LatticeFlow (XLA Fused Tensor, 2026) JAX Python 
@staticmethod
@jax.jit
def _hpp_rule(state):
    # UNPACK: Extract the 4 directional bit-planes from the single uint8 tensor
    n, s, e, w = (state >> 0)&1, (state >> 1)&1, (state >> 2)&1, (state >> 3)&1
    
    # EVALUATE: Determine head-on collisions across the entire grid simultaneously
    coll_ns = (n & s) & ~(e | w)
    coll_ew = (e & w) & ~(n | s)
    
    # INTERACT & REPACK: Apply self-inverse HPP swap via bitwise operations
    new_n = jnp.where(coll_ns, 0, jnp.where(coll_ew, 1, n))
    new_s = jnp.where(coll_ns, 0, jnp.where(coll_ew, 1, s))
    new_e = jnp.where(coll_ew, 0, jnp.where(coll_ns, 1, e))
    new_w = jnp.where(coll_ew, 0, jnp.where(coll_ns, 1, w))
    
    return (new_n << 0) | (new_s << 1) | (new_e << 2) | (new_w << 3)

The "Phase-Sync" Secret

To achieve the "Three Kings" Echo, reversing time is not just a matter of rolling the tensors backward. When the system switches direction, an extra standalone _hpp_rule must be fired. This un-collides the particles from the final forward step so they don't instantly shift away from each other in the first backward step, resulting in zero noise.

IV. Future Directions: Building the LatticeFlow Library

This endeavor is a foundation meant to be built upon. The ultimate vision is to establish LatticeFlow as a proper, open-source python library—the direct spiritual inheritor of the CAM-8 work and SIMPSTEP for modern hardware. We invite contributors to help build this roadmap:

  • Reviving the Archives: Porting all historical examples located in the simpstep/Examples directory into the modern JAX architecture.
  • The Cellular Automata Machines Revival: Re-coding the classic examples from Toffoli and Margolus's seminal CAM-8 manual into executable, browser-based tutorials.
  • API Formalization: Wrapping the Transport and Rule mechanics into a high-level API so users can define complex CA without managing the underlying bitwise JAX compiler ops.

Case Study: The Polymer Rule

As a preview of future complexity, historical SIMP code for polymer simulation (polymer.py) utilizes a 'pinwheel' partitioning CA neighborhood. Particles move only if doing so doesn't break polymer chains. Because particles cannot swap independently, the rule requires evaluating a 2x2 "together" block to respect the 'sensitive horns' of the dogbone geometry. Implementing this natively in the future LatticeFlow library will demonstrate the framework's capacity for multi-site partitioning logic on tensors.
